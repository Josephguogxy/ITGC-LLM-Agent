ITGC Agent System

A paper-aligned code release focused on the core multi-agent architecture: strategic planning, bridge construction, distributed dispatch, verification, semantic review, and execution feedback.

7-layer closed loop Solver-centered decisions GitHub Pages ready
Scope

What This Repository Keeps

Included

  • Agent roles and the orchestration framework
  • Long-term planning and short-term dispatch models
  • Optimization backends and solver schemas
  • LLM provider abstraction and prompt templates
  • Input contracts and minimal configs

Excluded

  • Experiment runners and batch scripts
  • Paper plotting pipelines and report generators
  • Generated outputs and result folders
  • Streaming demos and publication artifacts
  • Scenario packages that only support experiments
Flow

Closed-Loop System Logic

01

ITGC

Updates strategic weights and calls the long-term planner.

02

Orchestrator

03

DSE

05

Verification

Checks network feasibility and rejects unsafe schedules.

06

HCII

Injects semantic review and operator-facing reasoning.

07

Execution

Applies the accepted action and returns next-cycle state.

Code Map

Module Explorer

`src/agent/itgc_agent.py` + `src/models/long_term.py`

The strategic layer combines long-horizon objective reweighting with the planner wrapper. The main formal planning logic lives in the Benders-style planner under `src/optimization/long_term/`.

  • Strategic weight updates after each cycle
  • Long-term resource plan generation
  • Cut and iteration summaries for planning history

`src/agent/framework.py`, `orchestrator_agent.py`, `dse_agent.py`

This layer is the structural backbone of the repository. It threads the long-term plan, bridge variables, local state estimation, review, and feedback into one persistent weekly control loop.

  • Cycle orchestration and memory
  • Rollback logging and accepted-cycle tracking
  • State carry-over between rolling windows

`src/optimization/`

The numerical core is split into a problem builder, backend factory, formal long-term planner, and distributed short-term ADMM dispatch backend.

  • Bridge-to-problem transformation in `builder.py`
  • Distributed dispatch in `short_term/admm_dispatch.py`
  • Heuristic and formal backend compatibility

`src/llm/` + `src/agent/prompts.py`

The semantic layer stays provider-agnostic. Mock mode is preserved for reproducible reading, while OpenAI and Anthropic clients remain optional.

  • Unified request and response schemas
  • Provider factory for mock and real clients
  • Prompt templates for each agent role

`src/models/types.py` + `src/data/schemas.py`

These files make the repository readable. They define the objects shared across planning, dispatch, verification, and execution.

  • `LongTermPlan`, `BridgeVariables`, `ShortTermDecision`
  • `OperationalState` and `CycleFeedback`
  • `TimeStepData` and `StreamingSnapshot` input contracts
